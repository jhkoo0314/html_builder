# Backlog And Roadmap

작성일: 2026-03-01

## 1. 우선순위 기준
- P0: 보안/데이터 손상/서비스 중단 위험
- P1: 핵심 기능 안정성/운영성
- P2: 사용자 경험/개선 항목

## 2. Phase별 백로그

### Phase 0 (1주, P0)
- [ ] 업로드 제한 적용(file size/count/mime)
- [ ] 인증/인가 미들웨어 적용
- [ ] 레이트리밋 적용
- [ ] 상태/로그 API 접근 통제

### Phase 1 (2~3주, P1)
- [ ] 중복 parser/pipeline 공통 모듈화
- [ ] OpenAPI 명세 + 스키마 검증 도입
- [ ] 통합 테스트 구축
- [ ] CI 품질 게이트 적용

### Phase 2 (3~6주, P1/P2)
- [ ] 비동기 Job + Queue 도입
- [ ] 결과 저장소 표준화
- [ ] 관측성 대시보드 완성
- [ ] 운영 자동화(배포/롤백)

## 3. 의존성
- 인증 체계 확정 -> API 계약 확정 -> 클라이언트 반영
- 데이터 모델 확정 -> 마이그레이션 계획 확정
- 관측성 스키마 확정 -> 알람 정책 확정

## 4. 위험/완화
- 구조 변경으로 회귀 위험 증가 -> 계약 테스트 선도입
- 일정 압박 -> Phase 범위 엄격 관리
