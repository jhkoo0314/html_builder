# Migration Plan

작성일: 2026-03-01

## 1. 전략
- 빅뱅 전환 대신 점진적 전환.
- 신규 시스템은 `v1` 경로로 병행 운영 후 트래픽 스위칭.

## 2. 단계
1. 신규 API 구축 및 staging 검증.
2. read-only 관측 병행(미러링 가능 시).
3. 내부 사용자 canary(5%).
4. 점진 확대(25% -> 50% -> 100%).
5. 구 시스템 정리.

## 3. 데이터 마이그레이션
- 기존 아티팩트 유지 기간 정의(예: 90일).
- 필요한 메타만 이관, 원본 파일은 정책에 따라 보관/폐기.

## 4. 실패 대응
- 트래픽 즉시 롤백 스위치.
- 신규 job 생성 중지 후 기존 처리 유지.
- RCA 후 재시도.

## 5. 완료 기준
- 2주 연속 안정 운영.
- SLO 충족.
- 주요 고객 불만/장애 없음.
