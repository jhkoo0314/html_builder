# Operations Runbook

작성일: 2026-03-01

## 1. 운영 모드
- `dev`, `staging`, `prod` 분리.
- 환경별 URL/비밀키/리소스 분리.

## 2. 배포 절차
1. CI 통과 확인.
2. staging 배포 및 smoke test.
3. 승인 후 prod canary 배포.
4. 메트릭 확인 후 전체 전개.

## 3. 장애 대응 기본
1. 알람 확인(무엇이 깨졌는가).
2. 영향 범위 파악(사용자/기능/시간).
3. 임시 완화(트래픽 제한/롤백).
4. 근본 원인 분석(RCA) 및 재발 방지.

## 4. 주요 점검 명령(예시)
- 헬스: `/api/v1/healthz`
- 상태: `/api/v1/jobs/{id}`
- 메트릭: `/metrics`

## 5. 롤백 기준
- 배포 후 10분 내 실패율 급증.
- 지연시간 2배 이상 상승.
- 인증/권한 오류 다발.

## 6. 온콜 체크리스트
- [ ] 알람 티켓 생성
- [ ] incident channel 개설
- [ ] 임시 조치 실행
- [ ] RCA 일정 확정
